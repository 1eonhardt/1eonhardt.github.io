队列学习笔记

定义

队列(Queue)。又称先进先出(FIFO)线性表，也是一种受限线性表。只允许在表的一端插入，另一端删除。

  • 队首(Front)：允许删除的一端。
  • 队尾(Rear)：允许插入的一端。

基本操作：

  • 入队(Enque)：向队尾插入一个新元素
  • 出队(Deque)：从队首删除一个元素
  • 读取头元素(GetHead)：获取队首元素，但不删除

顺序队列

普通队列

  • 初始状态：front == rear == 0
  • 入队：队不满时，先给队尾元素赋值，再将rear加1
  • 出队：队不空时，先将队首元素取出，再将front加1

缺陷：空间利用不充分，可入队的元素是固定的，即表长。超过就会溢出。但如果有元素出队，数组前面空出的部分就浪费了。

循环队列

为了解决以上假溢出的情况，可将队列从逻辑上视为一个环，每当该溢出时，指针自动回到0。

  • 初始状态：front == rear == 0
  • 入队：rear = (rear + 1) % MAXSIZE
  • 出队：front = (front + 1) % MAXSIZE
  • 队长：(rear + MAXSIZE - front) % MAXSIZE
  • 队满：front = rear?要区分
    • 空一个位置出来，即队首指针在队尾指针的下一位置作为队满的标志。
    • 在类型中加一个数据成员size。
    • 类型中加一个数据成员tag。删除时设tag=0,则front == rear且tag == 0时则为队空。插入时设tag=1，则front == rear且tag == 1则为队满。

链式队列

双端队列

两端都可以插入删除的队列。

  • 输入受限的双端队列：只允许在一端进行插入删除，另一端只能删除
  • 输出受限的双端队列：只允许在一端进行插入删除，另一端只能插入

设输入序列为1,2,3,4
(1)由输入受限队列得不到的输出序列
(2)由输出受限队列得不到的输出序列
(3)两个都得不到的输出序列

令输入受限队列仅end1可输入，输出受限队列仅end2可输出。
对输入受限队列：
若仅从end1输出，则相当于队列。则由Catalan公式知有14种输出序列。
因为输入受限，
当输入序列只有3个元素时：输出序列只有6个，Catalan数为5，得不到的输出序列为312，借助end2，可以得到。
当输入序列有4个元素时：如果第一个元素不为4，则4插在后3个数中间或最后都是可行的。如果第一个元素为4，则序列中必定剩下321或123，无论从哪一段端出，都不可能输出2。
所以，输入受限队列得不到的输出序列为4213和4231。

对输出受限队列：
同理。如果输出的第一个元素为4，则1,2,3必定都已经输入了。无论从哪一端输入，1和2必定是相邻的。所以，输出受限队列得不到的输出序列为4132和4231。

两个都得不到的序列为4231
